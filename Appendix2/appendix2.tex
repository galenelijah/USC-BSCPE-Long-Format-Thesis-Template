\chapter{Software Quality Assurance (SQA) Framework}

This appendix provides a detailed overview of the automated testing suite developed for the USC Patient Information System (USC-PIS). The suite is designed to ensure system integrity, security compliance, and performance stability across the backend and frontend components.

\section{Testing Environment and Commands}

The testing suite is executed within the project's virtual environment. The standard command for running the comprehensive SQA audit is:

\begin{verbatim}
python3 manage.py test tests_unit_v2 tests_integration_v2 tests_performance_v2 --noinput
\end{verbatim}

\section{Core Testing Modules}

\subsection{Unit Testing (tests\_unit\_v2.py)}
Unit tests focus on individual components and logic in isolation. Key areas covered include:
\begin{itemize}
    \item \textbf{Authentication Logic:} Verification of MFA tokens, password reset flows, and session management.
    \item \textbf{Security Audits:} Validation of the \texttt{pgcrypto} encryption signals in the \texttt{patients} and \texttt{authentication} modules.
    \item \textbf{Data Validation:} Ensuring form constraints and model field requirements are strictly enforced.
\end{itemize}

\subsection{Integration Testing (tests\_integration\_v2.py)}
Integration tests verify the end-to-end clinical workflows, simulating real-world usage of the system (ACA-HSD-04F).
\begin{itemize}
    \item \textbf{Patient Registration to Consultation:} Validates the flow from creating a new patient record to recording a medical consultation.
    \item \textbf{Medical Certificate Issuance:} Ensures that certificate generation correctly pulls data from the fitness assessment module.
    \item \textbf{Health Campaign Notifications:} Verifies that campaigns triggered in the backend correctly populate the notification system for target student demographics.
\end{itemize}

\subsection{Performance Testing (tests\_performance\_v2.py)}
Performance tests measure the system's responsiveness and stability under various load conditions.
\begin{itemize}
    \item \textbf{Latency Benchmarks:} Measuring the response time for key API endpoints (e.g., Dashboard metrics, Patient search).
    \item \textbf{Concurrency Handling:} Testing the system's ability to handle multiple simultaneous requests without database locking or significant performance degradation.
    \item \textbf{Encryption Overhead:} Specifically measuring the time impact of on-the-fly decryption for large patient datasets.
\end{itemize}
